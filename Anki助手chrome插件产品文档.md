# Anki 助手 Chrome 插件——产品与技术规格（MVP v1.0）

> 目标：让用户在任意网页上一键把内容裁剪为高质量 Anki 卡片，**默认安全、可编辑、可回退**，并尽量减少打断式操作。

---

## 0. 范围与原则

* **范围**：Chrome MV3 扩展，内容抽取 → Q/A 自动识别 → 预览确认 → 与 AnkiConnect 交互入库；包含设置页与最小可用的错误处理与复原。
* **非目标（MVP 外）**：云端同步、团队协作、跨设备历史同步、在线 AI 服务（仅预留本地/离线 AI 兜底接口）。
* **设计原则**：

  1. **可控保真**：尽量保留原文结构与格式，同时进行最小必要的安全清洗；
  2. **先机器后人工**：多策略级联自动识别，**人工确认**永远是最后一步；
  3. **默认安全私有**：所有处理在本地完成；
  4. **可撤销与可复试**：失败可重试，成功可追溯；
  5. **少感知延迟**：解析在 content script 侧，下载与入库走后台队列。

---

## 1) 入口与交互流（默认）

### 1.1 触发方式

* **右键菜单**：选中网页内容 → 右键 → “发送到 Anki”。
* **快捷键**：全局组合键（建议 `Alt + A`，支持设置页自定义）。
* **备选**：设置页中的“从剪贴板导入”。

### 1.2 预览形态（页面内浮层，content script 注入）

* **布局**：

  * 左列：自动识别出的 **Q/A 列表**（卡片集合）；
  * 右列：**原文 HTML 渲染**（清洗后保真预览）。
* **卡片编辑能力**：

  * **合并/拆分**（多段落合并为一张卡，或把一张卡的多个块拆分成多卡）；
  * **批量选择**（多选后统一修改卡组/模板/标签或批量删除）。
* **操作区**：

  * 顶部：卡组选择 / 模板选择 / 标签输入（可下拉与自动补全）；
  * 底部：**“添加到 Anki”**（显示即将入库数量、去重结果统计、失败重试入口）。
* **状态提示**：

  * 顶部条状徽标：成功/失败/去重命中；
  * 连接状态：AnkiConnect 可用性（绿/黄/红）。

### 1.3 核心用户旅程（顺序描述）

1. 用户选中网页内容 → 触发（右键或快捷键）。
2. content script 克隆选区 DOM → 清洗 → 运行 Q/A 识别级联 → 生成候选卡片。
3. 弹出浮层预览（可编辑、合并/拆分、设置目标卡组/模板/标签）。
4. 点击“添加到 Anki”后：

   * 后台 service worker **下载图片**（按策略）并改写资源引用；
   * 与 AnkiConnect 交互（读配置/建卡组/去重/写入）；
   * 结果回传浮层并标记每张卡的入库状态；
   * 支持**失败项单独重试**与**导出错误报告**。

## 2) 获取内容：选区优先，剪贴板兜底

* **默认策略**：基于选区 DOM（`document.getSelection()` → `Range` → 克隆 DOM）抽取 **text/html 片段**。

  * 优点：保真度高、可获取结构化节点；对“禁用复制”站点更稳。
* **兜底开关**：

  * 设置页允许“从剪贴板读取”；
  * 浮层中提供按钮“从剪贴板导入”（用户主动触发）。
* **选区缺省**：若无选区，提示用户进行精确选择，或引导使用剪贴板导入。

---

## 3) Q/A 自动识别（多策略级联＋人工确认）

### 3.1 级联规则（默认全开，顺序从强到弱）

1. **显式标记**：匹配“Q: / A:、问：/答：、问题：/答案：”等（**可配置正则**，支持中英文与变体）。
2. **结构启发**：明显的标题/粗体/强调节点作为 **Question**，其后紧邻的第一个段落或块级节点作为 **Answer**。
3. **问号启发**：以“？”或“?”结尾的句子 → 下一段作为 **Answer**。
4. **列表/表格启发（可选）**：有序/无序列表中相邻两项，或表格首列/首行规则；默认关闭，避免误拆。
5. **失败兜底**：未命中任何规则时，将所选内容整体作为 **Front**，并提供 **快速框选工具** 让用户手动圈定 Q 与 A。

> **备注**：每条匹配产生**置信度**（强匹配权重大），UI 中以提示色标注；低置信度条目默认**不勾选**防止误入库。

### 3.2 多对识别与去误

* 连续多对问答可生成多张卡片；
* 默认逐条勾选，支持“全选/全不选”；
* 显示“本次将添加 X 张卡（去重后 Y 张）”的预估。

### 3.3 人工确认与快捷编辑

* 单卡片内支持：追加/删除段落、Markdown 快捷（粗体/斜体/代码等）。
* 批量：统一标签、卡组、模板；

---

## 4) HTML 保真与安全清洗

### 4.1 清洗器

* **默认**：DOMPurify + 自定义白名单。
* **保留标签**：`p, b, strong, i, em, u, s, code, pre, span, sup, sub, br, hr, ul, ol, li, blockquote, img, table, thead, tbody, tr, th, td, h1-h4`。
* **保留样式属性（精简）**：`font-weight, font-style, text-decoration, color, background-color, text-align`。
* **剔除**：`script/iframe`、所有 `on*` 事件、外链字体、`position: fixed` 等恶作剧样式；清除跟布局无关的类名与行内样式（保留语义样式）。

### 4.2 结构规则

* **列表与表格**：保留 `<ol start>` 起始序号与完整表格结构；移除纯布局类。
* **代码块**：统一标准化为 `<pre><code>…</code></pre>`。
* **数学公式**：

  * 优先保留 **LaTeX 源**（检测 KaTeX/MathJax 的 annotation）；
  * 若缺失，则保留可渲染的行内/块公式；
  * **不要**把公式强转为图片。

### 4.3 文本与空白规范化（用于去重）

* 去除不可见字符、合并多空格、全半角统一（NFKC）、统一标点（中英文引号、破折号等）。

---

## 5) 图片与媒体处理

### 5.1 默认策略

* **下载外链图片** → 存入 Anki 媒体库 → 将 `src` 改写为本地文件名。
* **文件名规范**：`qa_clip_{hash}_{i}.{ext}`（含原扩展名）。
* **链接解析**：相对路径基于页面 URL 解析为绝对路径；`data:` URL 直接入库。
* **执行通道**：后台 service worker 负责抓取，避免 CORS；Manifest 中加入 `host_permissions: ["<all_urls>"]`。
* **去重**：按二进制哈希对图片去重，复用已存媒体文件名。

### 5.2 可选开关

* “**不下载外链图片**，仅保留占位/原链接”；
* “**大图转 WebP**”（节省体积，默认关闭）；
* 最大下载并发（默认 3），失败重试策略（指数退避，最大 3 次）。

### 5.3 边界处理

* 懒加载与 `srcset`：解析最佳分辨率的实际资源；
* CSS 背景图：尝试解析常见内联样式，无法解析时提示“背景图未抓取”。

---

## 6) 与 AnkiConnect 的交互

### 6.1 基础 API（默认组合）

* 读取：`deckNames / modelNames / modelFieldNames`；
* 兼容：卡组不存在 → `createDeck`；
* 媒体：`storeMediaFile`；
* 去重：`findNotes`（对 **Front** 纯文本规范化后取哈希作为键）；
* 入库：`addNote(s)`（默认 `modelName: Basic`，字段 `{ Front: …, Back: … }`）。

### 6.2 连接与授权

* **首次连接引导**：调用 AnkiConnect 的 `/version` 探测；若失败，立即弹出引导，指导用户将 `chrome-extension://<EXTENSION_ID>` 加入 AnkiConnect 白名单。
* **连接参数（默认）**：端口 `8765`；请求超时 `3s`；失败自动重试 `3` 次（指数退避）。
* **失败状态分类**：

  * `unreachable`：无法建立连接（端口未监听或进程未启动）。
  * `unauthorized`：白名单未包含扩展 ID 或被拒绝访问。
  * `timeout`：请求超过超时时间未返回。
* **未启动**：显眼提示“Anki 未运行”，提供 **重试** 按钮与“静默重试”间隔（默认 3s）。

---

## 7) 卡组、模板、标签、去重

### 7.1 模板（默认）

* `Basic`（后续可扩展 `Basic (and reversed)` 与 `Cloze`）。

### 7.2 卡组选择

* 记忆用户上次选择；
* **推荐映射**（可配置）：按域名推荐，如 `zhihu.com → 知识碎片`。

### 7.3 标签模板

* 默认：`from-clip, source:{domain}, date:{YYYY-MM-DD}`；
* 设置页支持 **变量编辑** 与预览：`{domain} / {title} / {YYYY-MM-DD} / {HH:mm}` 等；
* 时区：随系统时区（例如 Asia/Taipei）。

### 7.4 去重策略（默认）

* **规范化 Front**：去除 HTML、统一空白与标点（含 NFKC 正规化）。
* **生成哈希标签**：计算规范化 Front 的 SHA-256，取前 12 个十六进制字符，写入标签 `hash:xxxxxxxxxxxx`（例如 `hash:abcd1234ef56`）。
* **查重流程**：

  1. **优先标签**：调用 `findNotes` 以 `tag:"hash:…"` 查询，命中即判为已存在；
  2. **历史兜底**：若命中文档缺少该哈希标签，则对 `Front` 字段执行**忽略空白/标点**的全文匹配以判重。
* **入库行为**：命中去重则在 UI 标注“已存在”，默认不再创建（可在设置中关闭去重）。

---

## 8) 权限与稳定性（MV3）

### 8.1 Manifest V3（最小权限）

* `permissions`：`storage, scripting, contextMenus, activeTab, commands`；
* `host_permissions`：`"<all_urls>"`（用于抓取图片与解析选区来源）；
* 架构：后台 `service_worker`，预览在 `content script`，图片抓取走后台。

### 8.2 性能策略

* 内容解析/清洗在 **content script** 侧（靠近 DOM，减少数据复制）；
* 大图下载 **限速/并发上限**（默认 3）；
* 入库 **批量队列**，失败可重试/导出错误报告；
* 内存与渲染预算：浮层节点数与图片预览做懒加载；
* 长选区渐进式渲染（先文字后图片）。

### 8.3 存储

* 用户配置使用 `chrome.storage.sync`（体积友好）；
* 临时解析结果与错误日志用 `chrome.storage.local`（可清空）。

## 9) 错误与边界

| 场景           | 症状/提示                       | 用户动作          | 系统处理                    |
| ------------ | --------------------------- | ------------- | ----------------------- |
| 无法识别 Q/A     | 顶部黄色条：“本次将仅作为单卡 Front/Back” | 手动框选/拆分       | 允许继续入库或取消               |
| 模板字段不匹配      | “当前模型不含 Front/Back 字段”      | 切换模型或配置字段映射   | 读取 `modelFieldNames` 引导 |
| 选区过大/复杂布局    | “建议精确选择内容”                  | 继续/取消         | 渐进渲染、建议拆分选区             |
| 图片跨域/CORS    | “部分图片未能下载”                  | 选择保留占位        | 标注失败项，可重试               |
| Anki 未启动/未授权 | 红色提示 + 重试                   | 打开 Anki/配置白名单 | 指南 + 静默重试               |
| 去重命中         | 卡片右侧显示“已存在”                 | 可强制创建（关闭去重）   | 默认不创建                   |
| 网络离线         | 入库失败                        | 稍后重试          | 队列内重试与本地缓存              |

## 10) 可选增强（非 MVP）

* **站点级策略包**：自定义正则组模板与结构规则；
* **AI 兜底分割**：优先离线/本地；
* **站点→卡组学习映射**：按使用习惯自动归档；
* 模板扩展：`Cloze` 与 `Basic (and reversed)` 自动化；
* 片段历史库：最近 20 次裁剪记录，可再次编辑重投；
* 快捷摘录：在右键菜单中列出最近使用卡组；
* 无障碍：键盘无障碍与屏幕阅读优化；
* 国际化：中英双语 UI 与日期本地化（含周起始日）。

## 11) 设置页（Options）

**分区与默认值**：

1. **基础**：

   * 默认卡组（记忆上次）；
   * 默认模板（`Basic`）；
   * 标签模板（`from-clip, source:{domain}, date:{YYYY-MM-DD}`）。
2. **识别规则**：

   * 显式标记正则（可编辑）；
   * 结构启发开关；
   * 问号启发开关；
   * 列表/表格启发（默认关）。
3. **HTML 白名单**：

   * 标签与样式属性（可视化多选）；
   * 代码/数学的标准化勾选；
   * 链接与外部资源处理策略。
4. **图片策略**：

   * 是否下载外链图片（默认开）；
   * WebP 转换（默认关）；
   * 下载并发/重试次数；
   * 超大图片阈值与降采样策略。
5. **数据与隐私**：

   * 所有处理本地执行；
   * 清空本地缓存/日志；
   * 导出诊断包（仅本机）。
6. **快捷键**：

   * 触发键。
7. **兼容性**：

   * 允许从剪贴板读取（默认关）；
   * 站点黑/白名单。

## 12) 可观测性与诊断（本地）

* **日志级别**：错误（默认）/信息（可选）/调试（开发）；
* **错误报告导出**：打包最近一次操作的入库流水、失败项、图片下载报告（含失败 URL 与状态）；
* **健康面板**（设置页）：

  * AnkiConnect 状态（可达/版本/白名单）；
  * 媒体存储写入测试；
  * 最近 10 条操作结果摘要。

---

## 13) 质量保障（QA）与验收标准

**用例集合（抽样）**：

* 文本结构：

  * 单问单答、连续多对 Q/A；
  * 仅问号启发可识别的段落；
  * 标题+段落形式；
  * 失败兜底并手动分割。
* 格式保真：

  * 列表、表格、引用、代码块、数学公式（KaTeX/MathJax 页面）；
  * 字体/颜色/对齐等样式精简后仍可读；
  * 复杂布局（多列/广告/粘性导航）。
* 图片与媒体：

  * 外链、相对路径、`data:` URL；
  * 懒加载、`srcset`、CSS 背景图；
  * 大图/动图下载与并发控制。
* AnkiConnect：

  * 未启动/未授权/白名单缺失；
  * 模型字段不匹配；
  * 去重命中与强制创建；
  * 批量 1/10/50 张卡片入库。

**验收标准（MVP）**：

* 90% 常见网页在默认规则下可得到**正确的 Q/A 候选**；
* 100% 入库前可在浮层中**人工确认**并编辑；
* 任何失败都有 **明确原因** 与 **可重试** 按钮；
* 默认策略下，**图片可正确入库并改写引用**；
* 去重生效且不会误删用户想保留的卡（可关闭去重）。
